## Refuted by adversarial verification (checked, NOT bugs)

These looked suspicious but the verifier proved them safe — useful to know they were examined:

- `today-drill:155` `.find(...)!` — `validDrillIds` is `drills.map(d=>d.id)`, so the find can only miss on an empty program, which `gatherInputs` guards against upstream.
- `sliderScoring` returning `{}` for unknown sliders — only ever called with `power_speed`; adding a slider is a code change that would surface immediately.
- `profile-view` switches lacking `default` — `string|null` return + `strictNullChecks` makes non-exhaustiveness a **compile error**, not silent.
- `games/hub` non-exhaustive view-kind, `save-session` body validation, `timeline` NULL-summary crash, `coach-progress` response.ok, `FighterCounterCard` LLM-field deref, questionnaire stale-closure setTimeout, `stress-test` retry, `save-session` sequential writes — all examined and refuted or judged non-issues.
- `debug-eval`/`test-rag` Voyage retry (#19) — downgraded to NONE (not wired into package.json; cheap idempotent hand-run scripts).
